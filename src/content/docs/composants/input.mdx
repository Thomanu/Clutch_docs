Champ de saisie. Hauteur 48px, fond blanc, **focus ring lime 2px** (et non violet), label flottant en « chip ». 5 états. La **Search Bar** n'est pas un composant séparé : c'est `Input type="search"`.

## Types

| Type     | Particularité                                    |
| -------- | ------------------------------------------------ |
| text     | Champ standard avec label flottant               |
| search   | Icône loupe leading + clear ✕ trailing auto       |
| dropdown | Chevron trailing, pas de saisie libre            |
| password | Œil trailing pour révéler le mot de passe        |

## Props

```ts
type?: 'text' | 'search' | 'dropdown' | 'password'
label?, placeholder?, value?, supportingText?: string
error?, disabled?: boolean
onChange?: (value: string) => void
onClear?: () => void
```

## Usage

**À faire**
- Toujours un **label OU un placeholder** explicite ; `supportingText` pour les règles de saisie.
- L'état `error` est **toujours** accompagné d'un message (jamais la couleur seule).
- Pour la recherche : `type="search"`. Ne pas recréer un champ ad hoc.
- Une seule recherche globale par écran (Sidebar ou Top Bar, pas les deux).

**À éviter**
- Créer une variante de recherche détachée du composant Input.
- Afficher l'icône clear si le champ est vide.
