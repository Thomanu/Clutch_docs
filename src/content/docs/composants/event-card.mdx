Carte d'événement de Clutch — annonce un match ou scrim à venir : date (jour + numéro), titre de l'événement, horaire et état de confirmation du roster (slots). **4 variants**, 1 axe State. Dimensions : 548×88px.

## Aperçu

<img src="/images/composants/event-card.png" alt="Event Card — états Open, Full, Live, Past" style="width:100%;border:1px solid rgba(26,25,28,.08)" />

## Variants

| Axe   | Valeurs                   |
| ----- | ------------------------- |
| State | Open · Full · Live · Past |

## Anatomie

1. **Date Block** (48×56px) : jour abrégé IBM Plex Mono Medium 10px, ink 40%, tracking 1 + numéro Space Grotesk Bold 24px, ink 100%.
2. **Content** (FILL) : titre Space Grotesk Bold 16px + méta IBM Plex Mono Regular 12px, ink 60%.
3. **Right Zone** : 5 carrés slots 10×10px + chevron › 14px ink 40%.

## État des slots

| State | Slots                        | Accent                     |
| ----- | ---------------------------- | -------------------------- |
| Open  | 4 lime + 1 rouge (manquant)  | `lime-100` / `error`       |
| Full  | 5 lime                       | `lime-100`                 |
| Live  | Slots + badge « EN DIRECT »  | `error` (rouge)            |
| Past  | 5 neutres atténués           | `primary-40`               |

## Props

```ts
state?: 'open' | 'full' | 'live' | 'past'
date: { weekday: string; day: string | number }
title: string
meta: string
slotsFilled: number
slotsTotal?: number   // défaut 5
onClick?: () => void
```

## Usage

**À faire**
- `State=Live` uniquement pendant le déroulement réel de l'événement.
- `State=Past` pour l'historique : contenu atténué, aucune action principale.
- Lister les Event Cards en colonne, gap 12px, triées chronologiquement.

**À éviter**
- Dupliquer l'état du roster ailleurs sur la carte — les slots font foi.
