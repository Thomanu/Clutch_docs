Ligne de tournoi de Clutch — résume une compétition dans un listing browse : identité (jeu, nom, organisateur), détails (format, date, prize pool, maps), jauge d'équipes inscrites et CTA d'inscription. **4 variants**, 1 axe State. Dimensions : **1120×140px**.

:::note
À ne pas confondre avec le **Tournament Card** (760×148px, carte dashboard de progression bracket) — ce sont deux composants distincts.
:::

## Variants

| Axe   | Valeurs                     |
| ----- | --------------------------- |
| State | Open · Full · Live · Closed |

## Anatomie (4 zones)

1. **Z1 — Identité** : Game Badge lime + nom du tournoi (Base Neue Extra Bold Oblique 18px) + organisateur (SG Regular 14px, ink 60%).
2. **Z2 — Détails** : 2 lignes format · date · prize pool · maps (SG Regular 14px).
3. **Z3 — Teams** : header « ÉQUIPES » + compteur inscrits/capacité + jauge dégradée + places restantes.
4. **Z4 — CTA** : Status Badge (état) + bouton d'action (selon State).

## Mapping State → CTA

| State  | Badge           | Bouton                    | Accent       |
| ------ | --------------- | ------------------------- | ------------ |
| Open   | « OUVERT » lime | « Rejoindre » (Primary violet) | `lime-100` |
| Full   | « COMPLET » neutre | « Liste d'attente » (désactivé) | `primary-40` |
| Live   | « EN DIRECT » rouge | « Regarder »           | `lime-100` fort |
| Closed | « TERMINÉ » neutre | « Résultats »           | `primary-40` |

## Usage

**À faire**
- Utiliser dans les vues listing/browse de tournois (largeur conteneur 1120px).
- `State=Open` : CTA violet actif — incite à l'inscription.
- `State=Full/Closed` : CTA désactivé — informer sans induire une action impossible.
- Cohérence obligatoire Status Badge ↔ State (même libellé, même couleur).

**À éviter**
- Cacher la jauge Z3 quand le tournoi est plein — c'est l'information clé de l'état Full.
- Utiliser Tournament Row pour la carte de progression bracket (→ Tournament Card).
